# REGRAS GIT

## Branches
- usar nome no formato: tipo/escopo-resumo
- tipos recomendados: feat, fix, refactor, docs, chore

## Commits
- um objetivo por commit
- mensagem clara e curta
- padrao sugerido: tipo(escopo): resumo

## Pull Request
- descrever objetivo, impacto e validacao executada
- se houve uso de IA, informar claramente

## Merge
- preferir PR pequeno e com validacao minima
- evitar merge de mudanca grande sem plano incremental
